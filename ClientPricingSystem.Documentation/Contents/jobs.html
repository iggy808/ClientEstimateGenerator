Command Line Jobs

Job Structure Overview

There is a .NET command line project within the CEG solution entitled CEG.Jobs. The command line project is incorporated into the CEG infrastructure, and therefore has the ability to access and alter database information, as well as any relevant models, dtos, etc. The purpose of the command line project is to compartmentalize and provide a convenient entry point for logic that is needed either periodically, sparingly, or at will.

Job Argument Format

Every job must have its own unique job argument, and this unique job identifier must be the first argument issued in the command when triggering the job. Throughout the following documentation, the unique job argument will be written immediately beneath the job name in bold. Unique job arguments follow the form: -jobname (the hypen is part of the command).

Note: Arguments following the initial job argument are permitted, but implementation of additional arguments requires additional development and job configuration.

Configuring And Running Jobs In Visual Studio

In order to run a job, the CEG.Jobs project must be started with a unique job argument passed as the project's first parameter. This unique job argument must be configured prior to running the job, and Visual Studio provides a UI for providing the job an argument.

Steps to configure a command line argument from within Visual Studio:

  1. Click on the CEG.Jobs project in the solution explorer.
  2. Press Alt+Enter to open the project's properties. Alternatively, right-click the project and select "Properties".
  3. Select "Debug" from the selections on the left side of the project's properites.
  4. Select the hyperlink text reading "Open debug launch profiles UI".
  5. Insert the unique job argument (including single hyphen) into text box beneath the "Command line arguments" header.

Jobs

Seed Test Database

-seedtestdb

The Seed Test Database job drops the database pointed to by the CEG.Jobs appsettings file, and reinstantiates it with fresh, properly related test data. Currently, the job populates the database with 50 clients, 10 vendors, 200 orders, and 600 order items. The data is generated using fakers, a core feature of the Bogus package. Bogus is a .NET package focused on generating helpful test data, and makes the generation process highly configurable and efficient.
